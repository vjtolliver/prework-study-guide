HTML

  • HTML is the frame of a webpage
  • HTML creates structure of a webpage and is made up of elements
  • Elements are a type of content on webpage
  • Elements can also be used to divide and describe a webpage
  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • The title element portion is featured in the web browser's title bar

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS: Cascading Style Sheet
  • CSS is how webpage content is shown
  • Inline CSS: style elements through the HTML file
  • Internal Style Sheet: uses the style element in the head section of a file and makes changes for the entire page
  • External Style Sheet: Most common way to use CSS and easier to manage. CSS rules are put in a seperate file and linked to HTML.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • GitHub Work Flow: Create a branch from the main branch. Add commits, or changes. Open a PR. Merge and Deploy
  • add -A: commits all files in the branch
  • -m: adds a message with your commit and provides description of changes
  • git pull origin main: brings up base branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
